Incorporate the HSBC's Design Thinking methodology

Get an overview and follow the process' phases that are aligned with the Design Thinking methodology. The Design Thinking process is a repeatable approach which gives 'creative confidence' and the right to experiment, test and prototype. It's also an end-to-end action guide that outlines the methods to use and outcomes at each stage.

Mobilise

HSBC meaning: Design thinking process starts with Mobilise, where we ensure our project has a clear unbiased problem statement/opportunity area, scope and that team is set up for success.

Cyber action

Plan the design process

  • Include human-centered design as a mandatory step in the project lifecycle.
  • Define UX/UI milestones, such as user research, prototype evaluations, and usability testing.

Empathise

The Empathise phase focuses on understanding users through research and observation. This helps gather insights about their needs, challenges, and behaviors.

Actions

  • Conduct user interviews and observations
  • Create user personas based on research
  • Document user journeys and pain points
  • Gather qualitative and quantitative data about user experiences
Explore tools

Define

In the Define phase, we synthesize the information gathered during the Empathise phase to clearly articulate the problem we're solving.

Key activities

  • Analyze research findings and identify patterns
  • Create clear problem statements
  • Define user needs and success metrics
  • Prioritize opportunities based on impact and feasibility
See methods

Ideate

The Ideate phase involves generating a wide range of creative solutions to the defined problem statement.

Techniques

  • Brainstorming sessions with diverse team members
  • Sketching and visualizing concepts
  • Exploring alternative approaches
  • Evaluating ideas against user needs and business constraints
Ideation tools

Prototype

In the Prototype phase, we create scaled-down versions of the product to investigate the solutions generated during ideation.

Approach

  • Build rough prototypes to test key concepts
  • Create wireframes and interactive mockups
  • Focus on core functionality rather than visual design
  • Prepare for user testing sessions
Prototyping guide

Implement

The Implement phase involves bringing your solution to life and preparing it for launch.

Key stages

  • Create detailed specifications and requirements
  • Develop the solution with regular quality assurance
  • Conduct final user testing before launch
  • Prepare documentation and training materials
Implementation checklist

Assess

The Assess phase involves measuring the effectiveness of your solution against the initial goals and user needs.

Evaluation methods

  • Collect user feedback through surveys and interviews
  • Analyze usage metrics and performance data
  • Compare results with initial success criteria
  • Document lessons learned and areas for improvement
Assessment framework

Optimise

The Optimise phase focuses on continuous improvement based on assessment results and evolving user needs.

Improvement strategies

  • Prioritize enhancements based on user impact
  • Implement incremental updates
  • Refine the user experience based on feedback
  • Consider scalability and future needs
Optimisation techniques
